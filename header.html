Welcome to Richard's whiteboard thing. To figure out what this does, open the test page in several tabs at once, and try editing in one tab. Then check the other.

New pages:

  • Random blank page
  • A page named: +"_" + (random postfix)
  • A page named exactly:
  • Anything you want, you don't have to use these forms.
Usage notes:
  • You can use any URL in this directory matching the pattern "./XXXXXXXXX.wb", ( ./[a-zA-Z0-9._+=-]+\.wb , actually) and you get an interactive whiteboard you can edit with multiple people.
  • Change the extension to ".txt" to get a static version.
  • Permanence: *Not* guaranteed to be permanent. However, it *is* statically stored on the server, so will survive past reboots. The delete-time right now is set at one year, if you want it deleted sooner, just erase all text.
  • No edit history is kept, so note there is no long-term (or short-term) guarantee here- anyone with your URL could erase your work. One workaround is to keep the static link open. Null files and files under 100 bytes may be deleted sooner.
  • The data/ directory will never be publicly browseable - so if your URL is secret, it is safe from all but the server admin.
